# MVP 스펙 (확정)

인터뷰에서 합의된 범위를 기준으로 한 최소 기능 제품 정의입니다.

## 제품 한 줄

개발자가 **localhost**에서 **PDF·텍스트**를 올리면 **즉시 요약·리포트**가 나오고, **업로드 문서 안에서만** 답하는 **로컬 웹 앱**.

## 범위 안 (MVP)

| 영역 | 내용 |
|------|------|
| 입력 | `.pdf`, `.txt` 업로드 |
| 출력 | 화면에 요약·리포트(섹션 구조) |
| Q&A | 자연어 질문 → **문서 추출 텍스트에 근거한** 답변만 |
| 실행 | 로컬 HTTP 서버, 브라우저 접속 |
| 데이터 규모 | 소형 파일 위주 가정 |
| 상태 | 세션/히스토리 **로컬 저장만** |

## 범위 밖 (이번 MVP 아님)

- 로그 파일 전용 파서·에러 스택 전문 분석 파이프라인
- 멀티 사용자·클라우드 동기화
- 업로드 문서 밖 일반 상식 위주 답변을 기본으로 하는 모드

## 수용 기준 (체크리스트)

1. PDF/txt 업로드 → 텍스트 추출 성공
2. 추출 텍스트 기반 요약·리포트가 UI에 표시
3. Q&A가 문서 밖 지식에 무분별하게 의존하지 않음(근거 없으면 불가 응답)
4. 로컬 저장 정책대로 세션/목록 유지
5. README 절차대로 localhost 기동 가능

## 비기능 (MVP 수준)

- **지연**: 소형 문서에서 체감 가능한 시간 내 1차 응답(구체 SLA는 구현 후 측정값으로 갱신)
- **보안**: 기본값은 로컬 전용; 외부 LLM 사용 시 키는 환경 변수·로컬만
